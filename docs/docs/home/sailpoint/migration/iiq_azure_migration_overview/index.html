üß© 1. What Actually Gets Migrated?

‚úî Migratable (with transformation)

  • Identities
  • Accounts
  • Roles
  • Entitlements
  • Group memberships
  • Application ‚Üí Entitlement mappings
  • Certification definitions ‚Üí Access Review equivalents
  • Identity lifecycle logic ‚Üí Azure workflows
  • Provisioning logic ‚Üí Azure SCIM provisioning

‚ùå NOT directly migratable ‚Äúas-is‚Äù

(Based on official documentation)

  • Workflows (IIQ XML cannot run in Azure)
  • Rules (BeanShell not supported in Azure)
  • Task definitions (No Azure equivalent)
  • Application schema (Azure app model is different)
  • Application provisioning policies
  • Identity warehouse internals
  • Request Center
  • Joiner/Mover/Leaver logic
  • Forms (Azure has no equivalent)
  • Certification campaigns (cannot be imported)

These must be rebuilt using Azure cloud components.

üß≤ 3. Extraction Phase (SPConfig + REST + SQL)

‚úî SPConfig Export ‚Äî For Configuration

SPConfig is required for extracting:

  • Applications (full definition)
  • Application schema
  • Workflows
  • Rules
  • Policies
  • Identity profiles
  • Managed attributes
  • Bundles
  • Forms
  • Lifecycle events
  • Correlation configs
  • Certification definitions
  • Task definitions

REST and SCIM cannot extract these.

Official API:

POST /identityiq/config/export

‚úî REST API ‚Äî For Identity + Operational Data

REST is used for:

  • Identities
  • Accounts (links)
  • Roles
  • Managed attributes (limited)
  • Work items

REST delivers JSON, perfect for quick POCs and UI display.

‚úî SQL ‚Äî Deep Relationship Extraction

SQL queries extract data REST does not fully expose:

  • Identity ‚Üí Account ‚Üí Entitlement mappings
  • Application schema
  • Entitlement metadata
  • Role ‚Üí Entitlement mapping

SQL is supported in read-only mode (as per official guidance).

üéõ 4. Transformation Phase

Azure does not understand IIQ’s objects.
So the transformation layer:

  • Converts IIQ roles ‚Üí Azure Groups / Access Packages
  • Converts entitlements ‚Üí Azure Groups / App Roles
  • Converts identities ‚Üí Entra ID schema
  • Converts workflow logic ‚Üí Logic Apps
  • Converts provisioning ‚Üí SCIM app provisioning
  • Converts approval flows ‚Üí Access Package policies

This is not automatic ‚Äî it requires a mapping engine.

‚úî Access Packages
