## ğŸ“˜ Basic Definitions (Beginnerâ€‘Friendly)

### ğŸ”¹ What is an *Application* in SailPoint?
An **Application** in SailPoint represents any external system from which you bring account data.  
Examples: Active Directory, Workday, Database, or a **CSV file** (DelimitedFile connector).

### ğŸ”¹ What is an *Authoritative Application*?
An **Authoritative Source** is a system that provides the **primary identity data** (source of truth).  
Examples: HR systems like Workday, SAP HR, Oracle HCM, CSV files with employee data.

In SailPoint:
- Only **one** authoritative application is used to *create identities*.
- The authoritative app **owns lifecycle** (joiner/mover/leaver events).

### ğŸ”¹ What is an *Account Schema*?
A Schema defines:
- Which attributes to read (employeeId, firstNameâ€¦)
- Which attribute uniquely identifies accounts
- Which attribute displays the name
- Which attributes are multiâ€‘valued

### ğŸ”¹ What is Aggregation?
Aggregation = Loading accounts from the application into SailPoint.
